## 🏗️ 三套独立认证系统

系统采用三套完全独立的认证系统，互不干扰：

```
┌─────────────────────────────────────────────────────────────┐
│                    浏览器 localStorage                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────────────┐  ┌─────────────────┐  ┌─────────────┐  │
│  │ admin-storage   │  │  auth-storage   │  │influencer_* │  │
│  │                 │  │                 │  │             │  │
│  │  平台管理员      │  │ 工厂老板/员工    │  │   达人      │  │
│  └─────────────────┘  └─────────────────┘  └─────────────┘  │
│         ↑                    ↑                   ↑          │
│   /admin/login          /login      /influencer-portal/login│
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 🔄 同时登录测试

由于三套系统使用不同的 localStorage key，你可以在同一个浏览器中：

1. 用管理员账号登录 `/admin/login`
2. 用工厂账号登录 `/login`
3. 用达人账号登录 `/influencer-portal/login`

**三个账号可以同时保持登录状态，互不影响！**

## ⚠️ 注意事项

1. **密码安全**: 这些是测试账号，生产环境请使用强密码
2. **数据隔离**: 不同工厂的数据是完全隔离的
3. **权限控制**: 商务人员无法访问工厂老板的功能
4. **认证隔离**: 平台管理员使用独立的认证系统
5. **测试数据**: 可以随意修改测试数据，需要时可重新运行 seed

## 🆘 常见问题

### 无法登录
- 检查是否使用了正确的登录入口
- 确认邮箱和密码是否正确
- 确认后端服务是否正常运行
- 查看浏览器控制台的错误信息

### 管理员登录提示"仅限管理员"
- 确保使用 `/admin/login` 入口
- 确保使用管理员账号而非工厂账号

### 重置测试数据
```bash
cd packages/backend
npm run db:migrate
npm run db:seed
```
